# 05 — UI System  
## EventPilot AI · Design System & Experience Guide

---

## Document Purpose

This document defines the visual language, interaction principles, and UI standards for **EventPilot AI**. The system is designed to create a premium, modern, elegant, and high-conversion event platform experience suitable for competitions, startup demos, and real-world deployment.

---

# 1. Product Design Tone

## Core Feel

EventPilot AI should feel:

- Intelligent, but not robotic  
- Premium, but not flashy  
- Modern, but timeless  
- Helpful, but not intrusive  
- Fast, calm, and trustworthy  
- Event-focused, energetic, and polished

## Emotional Experience

Users should feel:

- In control  
- Guided  
- Confident  
- Excited about the event  
- Impressed by the product quality

## Design Keywords

Elegant · Smart · Clean · Dynamic · Premium · Human-Centered · Sophisticated

---

# 2. Visual Personality

## Brand Personality

EventPilot AI blends:

- Startup-level innovation  
- Apple-like cleanliness  
- Notion-like clarity  
- Stripe-like polish  
- Modern event energy

## Interface Character

- Rounded but refined corners  
- Soft shadows with depth  
- Spacious layouts  
- Intelligent use of gradients  
- Minimal clutter  
- High-end dashboard feel

---

# 3. Color Direction

## Primary Palette

| Role | Color |
|------|------|
| Primary Brand | Deep Indigo |
| Accent | Electric Blue |
| Success | Emerald Green |
| Warning | Amber |
| Error | Crimson |
| Surface | Soft White |
| Background | Light Gray / Midnight Navy (dark mode) |
| Text Primary | Rich Charcoal |
| Text Secondary | Cool Gray |

## Recommended Style

- Use gradients sparingly for hero sections and premium CTAs  
- Keep core app surfaces neutral and clean  
- Use color for hierarchy, not decoration

## Suggested Premium Gradient

`Indigo → Blue → Violet`

---

# 4. Typography Direction

## Font Style

Use modern sans-serif fonts:

- Inter  
- Manrope  
- Plus Jakarta Sans  
- SF Pro style alternatives

## Hierarchy

| Use | Size |
|----|------|
| Hero Heading | 48–64px |
| Section Heading | 28–36px |
| Card Title | 18–22px |
| Body Text | 15–17px |
| Labels | 12–14px |

## Tone Rules

- Bold headings  
- Medium UI labels  
- Comfortable reading line-height  
- Minimal all-caps usage

---

# 5. Spacing & Layout Guidelines

## Core Spacing Scale

Use 8px grid system:

`4 / 8 / 12 / 16 / 24 / 32 / 48 / 64`

## Layout Principles

- Large breathing room between sections  
- Tight grouping within cards  
- Consistent left alignment  
- Clear content hierarchy

## Max Width

- Website Content: 1280px  
- Dashboard Area: 1440px  
- Reading Content: 720px

## Section Padding

- Desktop: 80–120px vertical  
- Tablet: 64px  
- Mobile: 32–48px

---

# 6. Card & Section Styling

## Card Design

Use cards heavily across platform.

### Card Style

- Radius: 20px to 24px  
- Soft shadow  
- Light border  
- White / glass background  
- Hover lift effect

## Card Content Pattern

1. Title  
2. Supporting text  
3. Meta row  
4. Action area

## Section Design

Alternate section styles:

- Clean white  
- Soft tinted neutral  
- Gradient hero areas  
- Glass dashboard zones

---

# 7. Button Styles

## Primary Button

Use for main CTAs.

- Filled brand gradient or dark solid  
- White text  
- Rounded-xl corners  
- Strong hover elevation

Examples:

- Start Planning  
- Get My Agenda  
- Explore Sessions

## Secondary Button

- Light background  
- Border outline  
- Dark text

## Ghost Button

- Transparent  
- Minimal hover tint

## Button Rules

- Minimum height: 44px  
- Horizontal padding: 18–24px  
- Strong focus states

---

# 8. Icons & Illustration Guidance

## Icon Style

Use clean line icons:

- Lucide  
- Heroicons  
- Feather style

## Icon Behavior

- 20px / 24px standard sizes  
- Use paired with text when needed  
- Never overload screens with icons

## Illustration Style

Use selective illustrations:

- Isometric venue scenes  
- Smart routing maps  
- AI helper character (subtle)  
- Event crowd moments  
- Minimal abstract data visuals

## Avoid

- Cartoonish art  
- Random stock vectors  
- Overly playful graphics

---

# 9. Animation & Motion Principles

## Motion Feel

Smooth, premium, intentional.

## Recommended Motion

- Fade + slide entrances  
- Hover lifts  
- Smart card transitions  
- Loading shimmer states  
- Drawer slide-ups

## Timing

- Fast UI reactions: 150ms–220ms  
- Standard transitions: 250ms–350ms

## Avoid

- Bouncy excessive motion  
- Long delays  
- Distracting background movement

---

# 10. Accessibility Guidelines

## Contrast

Meet WCAG AA minimum.

## Text Rules

- Body text minimum 16px preferred  
- Never use low contrast gray text on white

## Keyboard Support

- Full tab navigation  
- Visible focus rings  
- Skip-to-content support

## Screen Readers

- Semantic HTML  
- ARIA labels where needed  
- Proper landmarks

## Inclusive UI

- Large touch targets  
- Reduced motion support  
- Accessible route mode visible

---

# 11. Responsive Design Considerations

## Desktop (1200px+)

- Multi-column layouts  
- Rich dashboard cards  
- Split panels

## Tablet (768px–1199px)

- Two-column sections  
- Condensed nav  
- Responsive cards

## Mobile (<768px)

- Bottom nav or drawer menu  
- Single-column cards  
- Sticky key actions  
- Thumb-friendly interactions

## Mobile Priority Areas

- Session browsing  
- Navigation map  
- AI assistant  
- Alerts feed  
- Help center

# 13. Micro-Interactions

Include subtle polish moments:

- Button ripple or glow  
- Saved session confirmation toast  
- Smart recommendation badge pulse  
- Card hover transform  
- Smooth tab transitions

---

# 14. Dark Mode Direction

Dark mode should feel luxurious:

- Deep navy backgrounds  
- Frosted cards  
- Soft glows  
- Strong readability

Avoid pure black.
